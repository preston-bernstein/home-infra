# LLM Wiki as compiled layer above the RAG Engine

The vault's `wiki/` folder is a compiled, LLM-maintained artifact — the RAG Engine indexes wiki pages, not raw notes. Raw captures land in `_raw/`, are processed by an ingest agent (local model via Ollama broker) into cross-linked wiki pages, then deleted. The Vault Indexer excludes `_raw/` so only compiled pages reach the RAG Engine. This follows Karpathy's LLM Wiki pattern: the wiki is the persistent artifact; it compounds with every ingest rather than retrieving from raw fragments on each query. Tradeoff: adds ingest overhead and a schema to maintain; raw notes alone would be simpler to operate. Chosen because the RAG Engine (now MiniRAG — see ADR 0010) performs better on structured, cross-linked input than on unprocessed captures, and because retrieval quality compounds over time as the wiki grows rather than staying flat.
