# Cloudflare Tunnel over Tailscale for public service access

Current setup uses Tailscale as the only off-network access path — every device must run a Tailscale client to reach home services. The goal is off-network access to Public Services (LibreChat, HA, estate sales scraper) from phones and laptops without requiring a VPN client. Cloudflare Tunnel (`cloudflared` running on desktop) creates an outbound-only connection to Cloudflare, which proxies `*.houseoflight.dev` inbound — no open ports on the USG, no residential IP exposure. Tailscale stays for admin/SSH access and Internal Services; it is not replaced, only demoted from the primary user-facing access path. Alternative considered: expose services directly via USG port forwards — rejected because it exposes the home IP and requires certificate management on every service.
