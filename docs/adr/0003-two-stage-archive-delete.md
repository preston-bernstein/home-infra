# Two-stage archive-then-delete for stale vault documents

When a file disappears from the vault (deleted, renamed, or Syncthing hiccup), the indexer does not immediately delete it from LightRAG. Instead it sets an `archived_at` timestamp in local state and waits 30 days before calling `DELETE /documents/delete_document`. This protects against accidental deletes and transient sync gaps — a file missing from one nightly scan is not necessarily gone permanently. After 30 days the auto-delete runs on the next nightly pass. A `--cleanup` flag provides a manual override: lists all archived docs with age and prompts for confirmation before deleting. Immediate delete on first-missing-scan was rejected because Syncthing sync delays could cause false positives during vault reorganization.
