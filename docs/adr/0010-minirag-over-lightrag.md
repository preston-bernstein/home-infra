# Migrate from LightRAG to MiniRAG

LightRAG's indexing pipeline requires the LLM to produce structured JSON (entity + relationship extraction). HKUDS recommends 32b+ for reliable output; the desktop GPU (RX 9070 XT, 16GB VRAM) can fit qwen2.5:14b at Q4 comfortably but not 32b. At 8b (llama3.1:8b), extraction fails ~35% of documents, producing a sparse/incorrect entity graph and poor query quality. MiniRAG, from the same HKUDS lab, is designed explicitly for small SLMs — it does not require structured JSON extraction and achieves comparable accuracy. The API surface is near-identical to LightRAG (`/query`, `/documents`, same auth), making lightrag-mcp reusable. LightRAG is replaced rather than patched; no Phase 1 workarounds (chunk size, GLEANING, 32k context) are applied.
